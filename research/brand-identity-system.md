# Building a Company Brand Identity System from a Landing Page

**Obelus Labs Research Document #5**
**Date:** March 2026
**Category:** Brand & Design Strategy

---

## Overview

A brand identity system is the complete set of visual and verbal elements that define how a company presents itself across every touchpoint. For Obelus Labs, the landing page already establishes a strong design language: cyan accent on dark backgrounds, monospace and display typography, grid aesthetics, and the obelus (÷) symbol. This document covers how to formalize that language into a repeatable, scalable brand system.

---

## The Six Core Brand Assets

A complete brand identity system consists of six foundational assets:

1. **Primary Brand Mark (Logo)** — The main logo used in most contexts. For Obelus Labs, this centers on the ÷ (obelus) symbol, which carries both mathematical and typographic history.
2. **Secondary Mark** — A simplified version for small sizes, favicons, app icons, and social media avatars. The ÷ symbol alone works well here.
3. **Color Palette** — Primary, secondary, accent, and neutral colors with values defined across color systems (HEX, RGB, CMYK, Pantone).
4. **Typography** — A type system covering headlines, body, UI, and code contexts.
5. **Visual Extensions** — Patterns, icons, illustrations, data visualization styles, and textures that reinforce the brand.
6. **Brand Tone** — The verbal identity: how the brand sounds in writing, from marketing copy to documentation to error messages.

## Brand Guidelines Document Structure

A brand guidelines document should cover the following sections:

### Logo Usage
- Clear space requirements (minimum padding around the mark)
- Minimum reproduction sizes (print and digital)
- Approved color variations (cyan on dark, white on dark, dark on light for print)
- Prohibited modifications (stretching, rotating, recoloring outside palette)

### Color Specifications
- Define each color in HEX, RGB, CMYK (for print), and Pantone (for physical goods)
- Specify accessible contrast ratios (WCAG AA minimum)
- Document color application rules (e.g., cyan is accent only, never used for large background fills)

### Typography Rules
- Headline hierarchy (H1 through H4 with sizes, weights, line heights)
- Body text specifications
- Code/monospace usage rules
- Fallback stacks for web and system contexts

### Voice and Tone
- Define the brand personality (e.g., technical but approachable, precise, forward-looking)
- Provide do/don't writing examples
- Differentiate tone across contexts: marketing copy vs. documentation vs. social media

## Omnichannel Consistency

Every touchpoint must feel like the same brand:

| Channel | Key Brand Elements |
|---------|--------------------|
| **Website** | Full brand system — colors, type, grid, animations |
| **Social Media** | ÷ profile image, grid banner, cyan accent on dark |
| **Pitch Decks** | Dark slides, cyan accent, JetBrains Mono headers, clean data viz |
| **Email Signatures** | Name, Title, Obelus Labs, ÷ symbol, URL, cyan accent line |
| **Documentation** | Consistent code font, header styles, color usage |
| **GitHub README** | Brand-aligned badges, consistent formatting, logo at top |
| **Business Cards** | Dark card stock, cyan foil or print accent, minimal layout |

### Social Media Specifics
- **Profile image:** ÷ symbol, cyan on dark, consistent across Twitter/X, LinkedIn, GitHub, Discord
- **Banner/header:** Grid pattern background with cyan accent elements
- **Post templates:** Dark background, cyan highlights, JetBrains Mono for any code or technical text

### Pitch Deck Specifics
- Dark slide backgrounds (`#0a0a0a` or `#111111`)
- Cyan accent for key data points, chart highlights, and CTAs
- JetBrains Mono for headers and data labels
- Minimal text per slide (aim for six words per bullet maximum)
- Clean, geometric data visualizations

## 2025-2026 Brand Trends

- **Neo-minimalism** — Stripped-back design with intentional use of space, type, and color. Obelus Labs already aligns here.
- **Purpose-driven design** — Brand systems that communicate mission, not just aesthetics. Every visual choice should reinforce the company's values.
- **Adaptive/modular systems** — Brand elements designed as components that can be recombined across contexts rather than rigid templates.
- **Dynamic guidelines** — Living documents (Notion, Figma) rather than static PDFs. Guidelines that evolve as the brand evolves.
- **Motion as brand** — Consistent animation language (easing curves, durations, transition styles) as part of the identity system.

## ROI of Brand Consistency

- Consistent brand presentation across platforms increases revenue by up to 23% (Lucidpress/Marq study)
- 80% of consumers say they are more likely to recall brands with distinctive, consistent visual assets
- Brand consistency reduces design decision time for teams — everything references the system instead of starting from scratch
- Investors respond to professional, cohesive branding — it signals operational maturity
